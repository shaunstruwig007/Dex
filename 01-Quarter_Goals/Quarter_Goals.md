# Quarter goals — Wyzetalk (product)

**Purpose:** Anchor **weekly priorities** and discovery work to a small set of quarterly outcomes. Detailed account lists and € MRR stay on the [Essential launch](../04-Projects/Wyzetalk_Essential_Launch.md) page; this file is the **narrative + targets** layer.

**PDLC (how we deliver):** **Discovery → Design → Develop → Test** → deploy → feedback → **Discovery** again. Every meaningful **Now** or **Then** slice should have an explicit **discovery + design kick-off** before build-heavy commitment. Monday.com carries execution tasks; the vault carries intent, decisions, and PRDs.

---

## Q1 2026 (Jan–Mar) — closing out

**Theme:** Lock Essential **MVP scope**, PRD/AC baseline, migration narrative, and execution readiness.

| Outcome | Status | Notes |
|---------|--------|--------|
| Essential named; GA window defined | ✅ | Mid–late Apr → May 2026 |
| Full Requirements tree in vault + design sign-off | ✅ | Per launch doc |
| Federated remote app + payslip path + WhatsApp | 🔄 | In launch scope; **formal PRDs/AC immediately after GA checkpoint** (see tasks) |
| Migration roadmap imported (quarters, MRR) | ✅ | Launch doc + asset |
| ICPs, GTM one-pager, dependencies/RACI | 🔄 | On task list |

*Last week of Q1:* finish anything that **blocks** April GA (spec gaps, owners, comms).

---

## Q2 2026 (Apr–Jun) — launch + first migrations

**Theme:** **Ship Essential (MVP)** and **migrate 2–3 clients** onto the new packaging / operating model.

| # | Goal | Success looks like |
|---|------|---------------------|
| 1 | **GA Essential** | MVP live in market **by end of April / early Q2** (align comms + internal “GA” milestone); WhatsApp + payslip-related scope either **in GA** or **explicitly phased** with customer comms |
| 2 | **Client migrations** | **2–3 clients** through **substantive migration** (onboarded, cutover, or production use as defined per account) — reconcile with Q2 cohort (e.g. Nu, KFC, Kauai) via **steering** |
| 3 | **PRD debt post-GA** | **Payslip / WhatsApp** (and related AC) **written and agreed** where still missing — not an open-ended “later” |
| 4 | **Migration machine** | Owners, cutover criteria, steering cadence **live** (even if kickoff date flexes slightly vs “end May”) |

**Explicitly out of Q2 “must ship” (unless pulled in by a client):** **Scheduler service** and **celebrations** (birthdays, long service, anniversaries) — those sit in **Then** on the [roadmap section](../04-Projects/Wyzetalk_Essential_Launch.md#product-roadmap--now--then-2026); they still need **discovery + design** before they consume build capacity.

---

## H2 2026 (Jul–Dec) — scale, depth, “Then” delivery

**Theme:** More migration waves (per existing quarterly MRR table), **integrations / SSO-dependent** accounts, and **Then-phase** capabilities driven by a **scheduler** (platform **service**) enabling **scheduled messages/posts** and **personalised moments** (celebrations).

- **Monday.com:** high-level features stay the **backlog view**; each major item gets a **Discovery → Design** gate before it locks sprint scope.
- **End state for 2026:** Essential stable in market; **Then** items either **in discovery/design** or **in controlled rollout** — not “surprise scope” inside Q2 GA.
