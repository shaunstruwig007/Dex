The Friday Signal — the weekly cadence that grounds discovery.

An advisory, steerco-facing market brief, produced every Friday. It runs parallel to the idea→PRD pipeline — not inside it — and feeds discovery a single typed artefact. One page. Three movements. Safe for the board.

Audience: steerco (CEO escalates)

The voice — three movements, every time

What we see

Observation · sourced · dated

Concrete moves, named sources, dated. No interpretation yet. If a sentence here can’t cite a URL or a vault PDF, it doesn’t belong in this section.

Two independent mining-sector sources this week framed the same operational need in identical terms…

What it means for WT

Interpretation · advisor voice

What the observation implies for Wyzetalk specifically — ICP segments, competitive lane, narrative posture. Never alarm vocabulary (“at risk”, “crisis”, “losing”). Always advisor framing.

The mining sector is not waiting for a product pitch — it is articulating its own requirements…

What we propose

Action · for steerco / CPO

One concrete proposal per item — a one-pager, a positioning frame, a follow-up scan. Decisions stay with the CEO / CPO; the brief proposes, never decides.

We propose WT produce a single one-page mining-sector positioning document … before end of Q2.

Hard exclusions — what never lands in the brief

Excluded

Never in the Friday Signal

  • Client account names in risk framing. Book-of-business is CPO-internal. Account names appear only in neutral reference contexts (case-study candidates, ICP illustration).
  • Numbers from the client snapshot. 05-Areas/Companies/Wyzetalk_Clients/index.md is visibility: cpo-internal, dashboard_surface: false. Owned by a separate skill on a separate cadence.
  • Risk-alarm vocabulary. “At risk”, “threatens”, “crisis”, “losing”. The CEO may forward this to the board — tone discipline is non-negotiable.
  • Padded AI bullets. Exactly three sub-lenses per week. “No movement worth a line this week” is a valid complete answer.

The tone test

Before any item lands

“If this landed in a board pack with no other context, would it create clarity or panic?”

If panic, reframe or drop. The brief advising the CPO is the prototype of what WT-the-product aspires to offer its clients — advisor-voice, sourced, dated, propose-not-decide. If the brief ever stops feeling like advisor-voice, the skill is broken.

The staleness contract — how discovery uses this

>7 days old → discovery warns & asks for a re-run

/initiative-discovery-custom reads the latest *_friday_signal.md as a primary input. The contract is enforced on file mtime, not on calendar week.

  1. Fresh (≤7 days): discovery treats the brief as authoritative for market signal alignment. Quotes from it land in discovery.researchNotes and discovery.competitorSnapshot.
  2. Stale (>7 days): discovery flags felix_stale: true, prepends a one-line warning to discovery.research.summary, and drafts an open question asking the user to re-run /weekly-market-intel-custom before the sweep.
  3. Missing entirely: discovery does not fall back to scraping ingest/ raw — that’s a different skill’s job. It writes a missing-brief warning and continues with vault-only context.

This is why the cadence runs on its own schedule and not on the pipeline’s. Outside-in market reality refreshes weekly regardless of whether anyone has a new idea to discover.

Companion skills — what runs alongside, and what it does

/weekly-discovery-sweep-custom Monday morning. Iterates every card in the discovery column and re-runs per-card discovery research. Consumes the latest Friday Signal as input. Checks freshness; warns if stale.
/initiative-discovery-custom Per-initiative discovery research, on-demand. Phase 0 + Phase 3 read the brief. Consumes the brief. Enforces the >7-day staleness contract above.
/weekly-exec-intel Different artefact, different audience. Writes <date>_weekly_exec_brief.md from a separate template. Adjacent, not a substitute. Don’t stack with the Friday Signal in the same session — both re-read the same ingest.

Flags worth knowing

Defaults are sane — you rarely need flags. Same-day re-runs short-circuit on ingest freshness. The brief is idempotent on filename. Voice and exclusions are baked in.

  • --deep — monthly mode. First Friday of the month. Re-reads sector PDFs in Step 3 and writes a monthly industry-pulse digest under synthesis/monthly/.
  • --skip-ingest — debug only. Skips Step 0 and uses whatever is already under ingest/.
  • --dry-run — render the composed brief to chat only; no file writes. Useful for previewing voice changes.
