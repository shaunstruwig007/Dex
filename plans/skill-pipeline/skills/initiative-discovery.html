Skill in the pipeline · stage 02

Initiative Discovery — turn a one-line idea into evidence.

Reads the vault — ICP, exec roadmap, weekly market intel, customer meetings, competitor profiles, related PRDs — and produces a defensible discovery package the PRD author can build on. Confirms-relevance per grep hit so stale references never become live evidence.

Skill: /initiative-discovery-custom
Replaces: /moneypenny-custom
Stage: 02 · Discovery
Output: Product_ideas/<slug>_discovery.md

Where it sits in the pipeline

Pipeline stages 01 Idea → 02 Discovery → 03 PRD → 04 Critique → 05 spec_ready → 06 Design → 07 Build

What it does — six phases, ~2 minutes wall-clock

Phase 0

Load + staleness checks

Loads the input idea, hard-fails if System/icp.md is missing, reads the exec roadmap, checks whether the latest Friday Signal is <7 days old.

~10s · loads context

Phase 0.5

Exec-roadmap match

Is this initiative on the exec roadmap? If matched, exco has already validated the proposition — discovery’s job is to scope and de-risk, not re-litigate. If unmatched, flag as gap.

~5s · sets evidence severity downstream

Phase 1

Vault scan + relevance confirmation

Greps / queries the vault for keyword overlap. Confirms relevance per hit — discards stale, tangential, or out-of-scope-marker references.

~30–60s · produces filtered evidence list

Phase 2

Customer evidence harvest

Top-5 most-relevant meetings. Extracts direct quotes (text, person, date, source). Cross-checks named people against person pages. Identifies SMEs the PM should talk to before spec lands.

~30s · quotes + stakeholder list

Phase 3

Competitor + market signal alignment

Reads filtered competitor profiles and the latest Friday Signal. Captures one-liner positioning per near-neighbour. Downgrades confidence on stale profiles.

~20s · competitor table + signal cross-ref

Phase 4

ICP fit + segment match

Compares the idea’s user segments and problem to the ICP. Identifies which segment(s) it serves. Flags cross-segment disqualifiers as cautions + open questions.

~10s · one-sentence rationale

Phase 5

Solution patterns + candidate slicing

2–4 patterns for how others have solved this class of problem. Candidate slices — first-pass walking-skeleton + thickening slices for the PRD author to refine. Always 3–6 candidates.

~30s · patterns + candidateSlices[]

Phase 6

Compose + evidence-gap log + open Qs

Composes the discovery package. Evidence-gap log — explicit list of what discovery could NOT find. Open questions for every low-confidence field, contradiction, or gap.

~15s · writes Product_ideas/<slug>_discovery.md

Terminal

Write — idempotent

Writes to 06-Resources/Product_ideas/<slug>_discovery.md. Re-runs preserve any sections marked <!-- user-edited -->; overwrites the rest.

writes · idempotent on filename

Inputs and outputs — typed contracts

Always loaded before talking

Inputs · in load order

  • The input idea — one-line, brief file, or existing discovery file (under --rerun / --deepen).
  • System/icp.md — ICP segments + cross-segment disqualifiers. Hard-fail if missing.
  • System/exec_roadmap.md — primary evidence source for exec-direction matching. Logs gap if absent, continues.
  • 06-Resources/Market_intelligence/synthesis/weekly/<latest>_friday_signal.md — flags felix_stale: true if >7 days old.
  • Related PRDs in 06-Resources/PRDs/*.md — surfaces intersections.
  • Competitor profiles filtered to ICP near-neighbours.
  • Vault scan: 00-Inbox/Meetings/ last 90d · 05-Areas/People/ · 05-Areas/Companies/ · 06-Resources/Product_ideas/ · Market_and_deal_signals.md.

The discovery package

Output · consumed by /prd-author-custom

One markdown file with frontmatter + ten sections in fixed order:

  • Frontmatter: discovery_id, iteration, icp_version_at_run, exec_roadmap_priority, felix_signal_at_run, related_prds, ready_for_prd + blockers.
  • Problem statement — evidence-grounded paragraph, cited.
  • User segments — primary, secondary, out-of-scope.
  • ICP segment match — one sentence + cross-segment notes.
  • Exec-roadmap match — priority N or off-roadmap + background risks.
  • Stakeholders — named SMEs to talk to before spec.
  • Customer evidence — verbatim quotes, person, date, source.
  • Competitor + market signal — near-neighbour table + Friday Signal synthesis.
  • Solution patterns — how others solved this class; build/buy + anti-pattern when surfaced.
  • Candidate slices — first-pass walking skeleton + thickeners (3–6).
  • Evidence gaps + Open questions + Stale references discarded (audit trail).

The load-bearing rule — confirm-relevance per grep hit

Stale references are noise, not signal — the skill aggressively prunes

For every grep / QMD hit during the vault scan, the skill answers three questions before the hit becomes evidence: (1) Is this reference still current (last touched <6mo, or anchored to active PRD/project)? (2) Does the reference actually speak to this idea, or is it incidental keyword overlap? (3) Is the reference inside an out-of-scope marker (e.g. “Future phase — not for this cycle”)?

Answer-no on any → discard. Discarded hits land in the “Stale references found + discarded” section of the output for full audit transparency.

Added 2026-04-29 after the multi-language walkthrough surfaced a stale “future phase” reference in Scheduled_Content.md that misled discovery for an hour. The PM can override with a <!-- keep-stale --> marker if they really want to retain a citation; the skill respects that on subsequent runs.

How to invoke

/initiative-discovery-custom "<one-line idea>"           # cold start from a one-liner
/initiative-discovery-custom <brief-file>                # discovery from an existing brief / card / inbox idea
/initiative-discovery-custom --rerun <discovery-file>    # re-run; preserves sections marked <!-- user-edited -->
/initiative-discovery-custom --deepen <discovery-file>   # interactive pressure-test on the 2–3 weakest fields

Hard refusals — what the skill will not do

Refusals

Stops cold on these

  • No System/icp.md → hard-fail. Discovery without an ICP is opinion.
  • No problem statement supplied (cold start with empty / unintelligible input) → ask for a one-liner before continuing.
  • --deepen on a clean discovery (no low-confidence fields, no open questions) → “Nothing weak to pressure-test. Run --rerun instead.”
  • Stale reference re-confirmed without explicit override → discarded.

Out of lane

Will never do these

  • Author the PRD — that’s /prd-author-custom.
  • Run market intel from scratch — reads the latest Friday Signal; never falls back to ad-hoc scraping.
  • Author the ICP — System/icp.md is read-only to this skill.
  • Echo internal client-activity numbers / account names into the output — informs reasoning, never echoes.
  • Treat a stale “future phase” reference as live evidence — pruned by Phase 1 rule.
